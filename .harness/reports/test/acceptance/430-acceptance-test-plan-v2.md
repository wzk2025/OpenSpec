### 1.2 验收范围分级

| 级别 | 定义 | 说明 |
|------|------|------|
| **必验** | 看板标记 ✅ 的功能；或 🔵 功能中**已确认可运行**的部分 | 必须逐项验证通过；进行中的功能按交付物拆分，仅已交付可运行部分标记必验，未交付部分标记降级 |
| **可验** | 看板标记 🔵 < 50%，有部分交付物但尚未完整可用 | 验证已有交付物，记录完成度，不作为发版阻断 |
| **降级** | 看板标记 ⬜ 或 🔴 的功能；或 🔵 功能中**尚未交付**的部分 | 不纳入验收，记录延期原因 |
| **不验** | 330 存量功能且未被 430 修改 | 以 330 基线回归报告为准 |

> **拆分规则**：当看板任务为 🔵 部分完成时，由该任务 Owner 在验收启动前提供"已交付功能清单"，明确哪些子功能已可运行。验收级别以拆分后的子功能为单位判定，而非以看板任务为单位。

## 2. 验收策略

### 2.1 三层验收模型

```
┌──────────────────────────────────────────────────┐
│ 第三层：业务场景 E2E 验收（3 场景）                │  ← 验收委员会（LYY 主导）
│   B-01 网络群障 / B-02 指挥调度 / B-03 故障根因   │
├──────────────────────────────────────────────────┤
│ 第二层：跨模块集成验收                             │  ← 各组交叉验证
│   本体 → 融合 → 编排 → 执行 关键链路贯通           │
├──────────────────────────────────────────────────┤
│ 第一层：模块级功能验收                             │  ← 各组自验
│   按交付状态逐项确认                               │
└──────────────────────────────────────────────────┘
```

### 2.2 验收方法

| 验收类型 | 方法 | 交付物 |
|---------|------|--------|
| 功能验收 | 人工冒烟 + 自动化测试 | 截图 + 测试报告 |
| 集成验收 | 跨模块接口串联 | 集成验证记录 |
| E2E 验收 | 业务场景全流程走通 | 截图/录屏 + 验收签字 |
| 性能验收 | 基线比对 | 性能测试报告 |
| 安全验收 | 权限穿透测试 | 安全检查清单 |

### 2.3 验收判定标准

| 判定结果 | 条件 | 处理 |
|---------|------|------|
| 通过 | 功能符合验收标准，无 P0/P1 缺陷 | 签字确认 |
| 有条件通过 | 存在 P2 缺陷但不影响主流程 | 列入遗留清单，限期修复 |
| 不通过 | 存在 P0/P1 缺陷 | 驳回修复，重新验收 |
